The Nife Framework

A server-side web application support library for PHP.

What is it?

A framework for writing PHP web applications that follows sane programming conventions and generally stays out of your way.

Nife provides:

  • Action/view separation via Response objects.
  • Utilities for solving common problems correctly and succinctly.
  • A convention for structuring your project, compatible with Composer.

Does it do MVC?

Nife doesn't force you to follow any specific design pattern, but it's designed to help you build systems with an MVC-ish architecture.

What differentiates it from other MVC frameworks?

Nife doesn't shove its own way of doing things down your throat and doesn't bloat your project with useless code. For example, Nife doesn't include its own router/dispatcher mechanism because I've never seen a PHP routing library that was anything more than a verbose, inflexible pseudo-DSL to write if/else chains (and possibly a reverse lookup mechanism which nobody consistently uses, anyway). With Nife, you just write your if/else chain.

Nife is a 'framework' in a truer sense of the word. It provides conventions to follow and a few, well-defined interfaces and utility classes to help you follow them.

Is it compatible with X?

Yes. Nife is a plain old object-oriented PHP library, which means you can mix it with whatever other libraries and your own code you see fit.

Is it efficient?

If your web site/application is more than one page and is doing a few require_onces already, Nife adds very little additional overhead.

Nife's classes are designed to avoid doing unnecessary computations or waits, so in some cases may provide a performance improvement over fully-procedural PHP.

Why "Nife"?

This name follows a long tradition of naming frameworks after misspelled pieces of equipment and/or fish.
